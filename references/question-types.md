# Question Type Master Reference

Apply these rules together with the Synonym Rule and Needle Rule at all times.

---

## ⚠ MATCHING RULE
Each full test uses **exactly ONE** matching type. See SKILL.md.

---

## ⚠ UNIVERSAL COMPLETION RULES
Apply to Note Completion, Sentence Completion, Table Completion, Summary Completion:

1. **No duplicate answers** — no two questions in the same set may share the same answer word or phrase, even partially.
2. **Verbatim only** — copy exact words from the source. Never paraphrase.
3. **Word limit is absolute** — if answer exceeds the limit, rewrite the source sentence so the answer fits within the limit.
4. **Chronological order** — questions must follow the order the information appears in the source.

---

## ⚠ NOT GIVEN / NOT GIVEN Quality Rule
A NOT GIVEN statement must be on a topic the passage **does discuss** — but neither confirms nor contradicts.
- The topic must be plausible enough that a careless reader might mark it TRUE or FALSE.
- Do NOT create NOT GIVEN statements on topics completely absent from the passage — these are too easy.
- Correct NOT GIVEN: passage discusses X but never states whether X leads to Y.
- Wrong NOT GIVEN: passage never mentions X at all — obvious, too easy.

---

## ⚠ MCQ Distractor Rule
Every wrong MCQ option must:
- Come from real source text — never invented
- Be **eliminatable only by reading the passage** — not by general knowledge or common sense
- Be plausible on first reading — a test-taker who has not read carefully should be tempted

---

## Multiple Choice (Single Answer)
- One correct answer; three distractors all sourced from the passage
- Tests inference and global understanding — not simple detail retrieval
- Stem must be a complete sentence or meaningful question
- All wrong options must require passage reading to eliminate
- **EXTREME LANGUAGE BAN:** wrong options must never use absolute language (all, always, never, entirely, completely, solely, only, every, none) unless that exact wording appears in the source. Absolute language is a test-taking giveaway that allows elimination without reading.
- **Cross-paragraph synthesis:** at least one MCQ per passage should require connecting information from two non-adjacent paragraphs

**Answer format:**
```
| Q | Answer | Needle | Difficulty |
|---|--------|--------|-----------|
| 35 | B | "the exact sentence that proves B is correct" | 🟡 MEDIUM |
```

---

## Multiple Choice (Multiple Answers)
- Usually 2 correct answers out of 5 options; state exactly how many to choose
- All correct answers explicitly supported in source
- All wrong options sourced from the passage — never invented
- Each wrong option must be eliminatable only by careful reading

---

## True / False / Not Given (Reading — Factual)
- **TRUE:** statement agrees with a specific passage sentence (paraphrased)
- **FALSE:** statement directly contradicts a specific passage sentence
- **NOT GIVEN:** topic is discussed in passage but neither confirmed nor contradicted anywhere
- ⚠ NOT GIVEN ≠ "topic not mentioned at all" — the topic must appear but remain unresolved
- ⚠ NEVER mark FALSE just because something is not mentioned
- Distribution per set: ≥2 TRUE, ≥2 FALSE, ≥2 NOT GIVEN
- **Difficulty distribution within the set:** first 1–2 statements should be EASY (directly confirmable), middle statements MEDIUM (require paraphrasing), final 1–2 statements HARD (require careful elimination across multiple paragraphs)
- Reserved for Passage 1 and Passage 2 only

**Answer format:**
```
| Q | Answer | Needle | Difficulty |
|---|--------|--------|-----------|
| 1 | FALSE  | "exact sentence that contradicts the statement" | 🟢 EASY |
| 3 | NOT GIVEN | "passage discusses [topic] in para 2 but does not state whether [specific claim] — not mentioned anywhere" | 🔴 HARD |
```

---

## Yes / No / Not Given (Reading — Opinion)
- Tests the WRITER'S VIEW only — not facts
- **YES:** agrees with writer's stated or clearly implied view
- **NO:** contradicts writer's view
- **NOT GIVEN:** writer expresses no view on this specific point
- NOT GIVEN: writer discusses the general topic but never takes a position on this specific claim
- ONLY use in Passage 3. Never in Passage 1.
- Distribution per set: ≥1 YES, ≥1 NO, ≥1 NOT GIVEN

---

## Matching Headings ← MATCHING TYPE (max 1 per test)
- One heading per paragraph; always ≥2 extra headings in the list
- Headings test the paragraph's main idea — not a single detail
- Passage 2 and Passage 3 only

---

## Matching Features ← MATCHING TYPE (max 1 per test)
- Match named people/organisations to their stated views or findings
- Word list must have ≥2 extra options beyond number of questions
- Every entity in the word list must be named in the passage
- Options may be used more than once — state this explicitly if so

---

## Matching (General) ← MATCHING TYPE (max 1 per test, Listening only)
- Match items from two sets based on script content
- Word list must have ≥2 extra options
- Each match must be explicitly stated — never inferred

---

## Matching Sentence Endings ← MATCHING TYPE (max 1 per test, Reading only)
- Complete a sentence by matching its beginning to the correct ending
- ≥2 extra endings provided beyond the number of sentence beginnings
- Each completed sentence must be **grammatically correct and semantically coherent**
- Endings must be specific enough to be unambiguous — avoid endings that could complete multiple stems
- Tests cause-effect, chronology, and logical relationships
- At least one sentence beginning should require understanding the relationship between two paragraphs to select the correct ending

---

## Sentence Completion
- Complete a sentence using exact words from the source
- ALWAYS state the word limit: "NO MORE THAN TWO WORDS" or "ONE WORD ONLY"
- Answers must be **verbatim** from source — never paraphrase
- Completed sentence must be grammatically coherent
- **No duplicate answers within the same set**

---

## Note Completion
- Fill gaps in notes organised under headings
- ALWAYS state the word limit: "NO MORE THAN TWO WORDS AND/OR A NUMBER"
- Answers **verbatim** from source
- Questions in chronological order of source
- **No duplicate answers within the same set**

---

## Summary Completion (From Text)
- Fill gaps using words taken directly from the source
- ALWAYS state the word limit
- **No duplicate answers within the same set**
- Summary may cover one section or the whole passage

---

## Summary Completion (From List)
- Fill gaps using words from a provided word list
- Word list must have ≥2 extra options beyond the number of gaps
- Words in the list may serve as distractors — include plausible wrong options
- **No duplicate answers within the same set**

---

## Table Completion
- Fill gaps in a table organising source information
- ALWAYS state the word limit
- Answers **verbatim** from source
- **No duplicate answers within the same set**
- Table tests ability to locate specific data and understand categories
